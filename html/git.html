Набори команд для GitHub

Завантаження Git

  1. Перейти на офіційний сайт Git git-scm.com
  2. Натисни кнопку Download for Windows
  3. Завантаж .exe файл на свій комп'ютер.
  4. Знайти завантажений .exe файл.

Встановлення Git

  1. Відкрити завантажений .exe файл.
  2. Прийняти умови ліцензійної угоди.
  3. Залишити стандартну директорію для встановлення (рекомендовано!)
  4. Залишити всі запропоновані інсталятором варіанти за замовчуванням, нічого не змінюючи.
  5. Натисни Install для завершення встановлення.

Перевірка встановлення

  • Запусти термінал на комп'ютері
  • Ввести команду git --version або її скорочену версію git -v
  • Натиснути Enter, щоб її виконати.

    • Якщо з'явиться номер версії, значить встановлення пройшло успішно.

Термінал Git Bash на Windows

! Під час встановлення Git також встановлюється термінал Git Bash На відміну від вбудованого у Windows PowerShell, він підтримує всі можливості терміналів операційних систем macOS та Linux.

Запуск Git Bash на Windows

  1. Перейти в меню Start (укр. Пуск) або шляхом пошуку в панелі завдань ввести Git Bash у пошукове поле.
  2. Клікнути на знайденому застосунку для запуску програми.

або

  1. Відкрити провідник Windows (Windows Explorer).
  2. Перейти в директорію, де ти хочеш працювати, наприклад, у папку твого проєкту.
  3. Клацнути правою кнопкою миші в контекстному меню і вибрати Show more options (укр. Показати більше опцій).
  4. У новому контекстному меню вибрати Open Git Bash here (укр. Відкрити Git Bash тут).
! Залежно від версії операційної системи випадаюче меню та розташування пункту Open Git Bash here може відрізнятися.

Платформа GitHub

GitHub доповнює Git, надаючи віддалене сховище для коду, полегшує командну роботу та гарантує надійність і безпеку.

Створення репозиторію на GitHub

Фізично репозиторій — це просто папка на комп'ютері, яку використовує система контролю версій Git.

Щоб створити новий репозиторій:

  1. На сайті GitHub у верхньому правому куті натиснути на кнопку +, щоб відкрити випадаюче меню. Вибрати пункт New repository
  2. В обов'язковому полі Repository name ввести назву свого репозиторію, наприклад, my-first-project.
  3. У необов'язковому полі Description можна додати короткий опис репозиторію. Це допоможе іншим розробникам зрозуміти, про що твій проєкт.
  4. Також можна вибрати, чи буде репозиторій публічним (Public), чи приватним (Private).
  5. Рекомендовано в полі Initialize this repository with: встановити галочку на чекбокс Add a README file. Це автоматично додасть у репозиторій файл, який можна заповнити інформацією про проєкт.
  6. У полі Add .gitignore рекомендовано обрати зі списку пункт Node.
  7. Натисни кнопку Create repository, і готово!

Віддалені та локальні репозиторії

Віддалені репозиторії (remote repository)

Віддалений репозиторій — це проєкт (папка) на сервері GitHub, де розробники зберігають код проєкту і діляться ним. Це дає можливість багатьом розробникам працювати над одним проєктом.

Локальні репозиторії (locale repository)

Локальний репозиторій — це копія віддаленого репозиторію (папка) но компьютері розробника, до можна вносити всі потрібні зміни а потім синхронізувати їх з віддаленим репозиторієм.

Налаштування SSH ключа

SSH-ключ потрібен для того, щоб працювати з проектом зі свого копьютера, а не через вебінтерфейс, використовуючи логін та пароль.

SSH-ключ — це унікальний ключ доступу до облікового запису на GitHub, який треба генерувати для кожного пристроюна якому створюєтся локальний репозиторій.

Генерація SSH-ключа

  1. Відкрити термінал (Git Bash) на своєму комп'ютері.
  2. Ввести наступну команду, де замість your_email@example.com потрібно вказати свій email
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
    Ця команда створить новий SSH-ключ та пов'яже його з електронною поштою. Потім буде запропоновано вказати місце збереження ключа (рекомендовано залишити запропонований шлях).
  3. Натисни клавішу Enter, щоб підтвердити адресу.
  4. Після цього буде запропоновано ввести кодову фразу (passphrase), яка буде аналогом пароля до SSH-ключа. (рекомендуємо нічого не вводити і натискати Enter).
    ! Якщо додати пароль ключа, то його потрібно буде вводити кожен раз під час роботи з GitHub, це не зручно.

Додавання SSH-ключа до ssh-agent

  1. Запусти програму ssh-agent через термінал командою:
    eval "$(ssh-agent -s)"
  2. Додати свій SSH-ключ до ssh-agent командою ssh-add:
    ssh-add ~/.ssh/id_rsa
! Шлях до SSH-ключа ~/.ssh/id_rsa потрібно буде коригувати в усіх командах на свій, якщо його було змінено під час генерації SSH-ключа.

Щоб переглянути доданий SSH-ключ, треба виконати команду:

cat ~/.ssh/id_rsa.pub

Додавання SSH-ключа до GitHub акаунту

  1. Скопіювати вміст SSH-ключа під час перегляду його вмісту за допомогою команди cat ~/.ssh/id_rsa.pub і клавіш Ctrl + C
  2. Перейди в налаштування SSH та GPG ключів у своєму профілі на GitHub.
  3. Натиснути на кнопку New SSH key, потім додати скопійований в п. 1 ключ, дати йому назву і натисни Add SSH key.
! До одного акаунту GitHub можна підключити декілька SSH-ключів. Тому якщо розробник працюєш з декількома пристроями, ттреба додати в налаштуваннях GitHub стільки SSH-ключів, скільки потрібно. У майбутньому завжди можна видалити зайвий SSH-ключ у налаштуваннях.

Клонування віддаленого репозиторію

Клонування — це процес створення локальної копії віддаленого репозиторію на своєму комп'ютері.

  • на сторінці репозиторію на GitHub скопіювати адресу репозиторію з розділу SSH.
  • відкрити термінал у папці Projects.
  • у терміналі виконати команду git clone, після пробілу додавши адресу репозиторію, який треба склонувати.
  • Натиснути Enter.

Створюємо живу сторінку (адрусу сайту) на GitHub

  • На сторінці репозиторію натискаємо на Settings
  • З правого боку в меню натискаему Pages
  • В Branch. Your GitHub Pages site is currently being built from the main branch, обераємо гілку main, та натискаємо Save.

Скасувати останній коміт, зберігаючи зміни в робочій директорії:

git reset --soft HEAD~1

Скасувати останній коміт і зміни в індексі (стейджинг), зберігаючи їх у робочій директорії:

git reset --mixed HEAD~1

Скасувати останній коміт і всі зміни, видаливши їх з робочої директорії:

git reset --hard HEAD~1

Відправка змін на GidHub з консолі

  • Використай команду git status, щоб перевірити статус локального репозиторію
  • Введи команду git add --all , щоб додати всі зміни під контроль Git
  • Повтори команду git status, щоб переконатися, що зміни під контролем
  • Введи команду git commit -m “add index file”, щоб зберегти зміни в історії
  • Введи команду git push, щоб відправити зміни на GitHub

  9. Натискаємо на шестерню зправа від About, обираємо пташкою "User you GidHub Pages website", натискаємо "Save changes"

Приклад реальної роботи з GitHub

Нам дали задачу зробити меню, ми це зробили і закомітили.

Перший рік, скорійш за все, ми не будемо маржити в гілку мейн самостійно.

Як ми працюємо на командному проекті:

  • Ми запушуємо зміни в нашу гілку
  • Заходимо на репозиторій, де буде повідомлення, що наша гілка була тільки що запушена, і зелена кнопка Compare & pull request.
  • Пулл реквест, це запит до тім-ліда, на те, щоб він перевірив наш код, і якщо він правильний, прийняв його в основну гілку.
  • Обираємо гілку, перевіряємо що конфліктів немає, з якої в яку гілку ми хочемо помістити код, і натискаємо кнопку Create pull request.
  • Гарно його підписуємо.
  • Можно вибрати людей які будуть перевіряти мою частину коду, натискаємо кнопку Create pull request.

Перед створенням нової гілки, для роботи на проекті, треба клонувати репозиторій проекту собі на комп.

Далі операції які на проекті тімлід виконує самостійно

Як зайти в гілку яку створив колега, та подивитись що у нього.

git pull - отримуємо останні зміни з сайту github.

git checkout "branchName" - переходить на гілку з назвою "branchName".

git add та git commit, ми можемо записати однією командою

git commit -am "text commits" - переходить на гілку з назвою "branchName".

git push - відправляємо на сайт github.

Щоб зробити якісь дії в гілці, треба в ній знаходитись.

Якщо у нас, наприклад був відкритий файл index.html, і в гілці в яку ми переходимо теж є файл index.html, то у нас буде відкритий файл index.html тої гілки, в яку ми перейшли.

Як зміни які ми робили в своїй гілці зарелізити в основну гілку (після перевірки ментором).

Ми маємо знаходитись в тій гілці в яку ми хочемо зробити marge.

git checkout main - зливаємо гілку з назвою "branchName" в поточну гілку.

Перевіряємо, чи дійсно ми в гілці main

git branch

git marge "banchName" - зливаємо гілку з назвою "branchName" в поточну гілку.

У нас залишаєтся код в гілці main, і до нього додаєтся код який ми зробили.

Після marge нам необхідно сайт опублікувати в інтернеті.

Якщо ми забули, що нам теба робити, ми вводимо команду:

git status - видає підказку, що до наступної команди.

git push .

git marge "banchName" - зливаємо гілку з назвою "branchName" в поточну гілку.

Як видалити гілку.

Наприклад, ми замерчили з гілки, і вона нам більше не потрібна. Нам треба виділити дві гілки публічну і локальну.

  1. Треба знаходитись не в тій гілці яку ми збираємось видаляти.
  2. Видалити гілку з репозиторію на GitHub.

git branch -b "branchName" - видалення гілки з компьютера.

git branch --delete origin "branchName" - видалення гілки з репозиторію.

git push origin --delete "branchName" - видалення гілки з репозиторію другий спосіб.

Перед кожним marge робити перевірку командою git status. Всі зміни треба зберігти перед marge. (Робимо коміт та пуш).

Далі маємо знаходитись не тій гілці яку мерджім, і після цього merge потрібну гілку.

Конфлікти merge та їх вирішення

Можно відкотити назад власний merge.

git merge --abbort - відміняємо merge (злиття гілок).

Дві частини коду, які виділені різними кольорами, зверху над конфліктуючими блоками 5 варіантів дій, з яких можно вибрати одну. Ось ці варіанти:

  1. Accept Change. Прийняти зміни.
  2. Accept Incoming Change. Прийняти вхідні зміни.
  3. Accept Both Changes. Прийняти обидві зміни.
  4. Compare Changes. Порівняти Зміни.
  5. Start Live Share Session. Почати сеанс прямого ефіру.
  • git status
  • git add . або git add file-name.ex;
  • git commit -am "text commits"
  • git push

Після того, як ми пофіксили конфлікт, нам треба ці зміни зафиксувати (додати add, підписати commit, відправити push на GidHub).

Потім видалити непотрібн гілку локально на компьютері, та на GitHub.

git branch -b "branchName" - видалення гілки з компьютера.

git branch --delete origin "branchName" - видалення гілки з репозиторію.

git push origin --delete "branchName" - видалення гілки з репозиторію другий спосіб.

Відправити зміни в буфер обміну, та завантаження з буферу

git slash - зберігаємо не збережені зміни в файлах і кладемо їх в буфер обміну.

git stash apply - вставляємо збережені змни з буфера обміну. Зміни в буферу обміну зберігаются до того моменту поки проект зберігається на компьютері. Вставити збережене можна в будь-який файл, будь-якої гілки.

git diff - показує зміни, яки були внесені в код після останнього коміту. Поки є внизу дві крапочки, можемо скролити зміни за допомогою клави (стрілочки вверх - вниз).

Щоб вийти з режиму перегляду змін, треба натиснути клавішу "q".

Подивитись історію коммітів в якійсь branch

git log

Натискаємо "q" щоб вийти з цього, та взагалі будь якого режимую.

So that the commit disappears but all the changes we made remain. After that, the commit will disappear and the files will be marked as modified. Так, щоб зобов'язання зникло, а всі зміни, які ми зробили, залишилися. Після цього після цього коміт зникне, а файли будуть позначені як змінені.

git checkout "branchName" - переходить на гілку з назвою "branchName".

git checkout -b "branchName" git checkout -b header - створює гілку з назвою "branchName" і переходить на неї.

git add . - зберігає зміни в файлах.

git commit -m "commit message" - підписуємо збережені зміни в файлах.

git push - відправляємо на сайт github.

git pull - отримуємо останні зміни з сайту github.

git status - показуємо статус проекту.

git branch - показує список гілок в проекті.

git branch -r - показує список гілок на сайті github.

git branch -a - показує список гілок на компьютері та на сайті github.

git fetch - отримуємо зміни з сайту github.

git marge "banchName" - зливаємо гілку з назвою "branchName" в поточну гілку.

git marge --abbort - відміняємо зливання гілок.

git push --set-upstream origin header або git push -u origin header - створюємо на github гілку і запушуємо код. Ця команда виконуєтся один раз після створення нової гілки. Надалі файли запушуються без цієї команди.

git branch -d branch-name - видалення гілки яка ще не замарджина на GitHub.

git branch -D branch-name - примусове видалення гілки яка ще не замарджина на GitHub.

Щоб видалити репозиторій з GitHub...

  1. Увійдіть до свого облікового запису GitHub.
  2. Перейдіть на сторінку репозиторію, який ви хочете видалити.
  3. Відкрийте вкладку "Settings". Вона знаходиться на панелі вгорі репозиторію, поряд з іншими вкладками, такими як "Code", "Issues", "Pull requests" тощо.
  4. Прокрутіть сторінку вниз до розділу "Danger Zone".
  5. Натисніть кнопку "Delete this repository".
  6. Підтвердіть видалення. Вам буде потрібно ввести назву репозиторію для підтвердження видалення. Після цього натисніть кнопку "I understand the consequences, delete this repository".
